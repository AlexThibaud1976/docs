# Your First Report

This guide walks you through generating your first PDF report in under 5 minutes.

<Steps>
  <Step title="Open TestPulse">
    In Azure DevOps, open your project, click **Test Plans** in the left sidebar, then click the **TestPulse** tab.

    You'll see four tabs at the top: **Report**, **History**, **Settings**, and **User Guide**.
  </Step>
  <Step title="Select a Test Plan">
    In the **Report** tab, click the plan dropdown and select your test plan.

    The suite tree loads automatically. All suites are selected by default — uncheck any suites you want to exclude from the report.

    <Tip>
      If the dropdown is empty, click the **Refresh** button next to it. Only plans in the current project are shown.
    </Tip>
  </Step>
  <Step title="Fill in the project information">
    Complete the form on the right:

    | Field | Notes |
    | --- | --- |
    | Report title | Displayed on the cover page |
    | Environment | DEV / QA / UAT / PREPROD / PROD |
    | Project reference | Your project code or ticket number |
    | Tester | Type 2\+ characters for ADO autocomplete |
    | Approver | Type 2\+ characters for ADO autocomplete |
    | Report date | Pre-filled with today's date |

    All visible fields are required unless marked optional. You can hide fields you don't need in [Settings → Field Configuration](/testpulse/settings-fields).
  </Step>
  <Step title="Click Generate">
    Click the **Generate report** button.

    A progress bar shows the data being fetched from ADO (test points, results, work items, attachments). This usually takes 5–30 seconds depending on the size of your test plan.

    If any required fields are missing, a list of errors appears — fill them in and click Generate again.
  </Step>
  <Step title="Export your report">
    Once generated, the report appears in the panel on the right.

    Click **PDF** to download a paginated PDF, or choose **HTML**, **Word**, or **Excel**.

    The report is also automatically saved to your **History** tab for future reference.
  </Step>
</Steps>

## What your report contains

By default, a generated report includes:

- **Cover page** — project info, date, environment, tester, approver
- **Executive summary** — automated verdict (Pass / Warning / Fail) with strengths and recommendations
- **KPI cards** — pass rate, execution rate, total cases, passed/failed/blocked counts, linked bugs
- **Results table** — latest outcome per test case, with suite, duration, tester, and date
- **Traceability matrix** — Epic → User Story → Test Case → Bug links
- **Bugs section** — all bugs linked to test cases in the selected plan
- **Charts** — pass/fail donut, results-by-suite bar chart

You can enable or disable each section from [Settings → Export Sections](/testpulse/settings-exports).
