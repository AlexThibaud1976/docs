# Changelog

All notable changes to TestPulse are listed here in reverse chronological order.

---

## v2.7.6 — 2026-06-09

**Internal quality release — 109 new tests**

- 109 new Vitest unit tests covering Word, UserPreferences, migration v274, Excel, PDF and HTML services
- 250 tests total across the project
- No functional changes

---

## v2.7.5 — 2026-06-01

**Export button visibility control**

- New setting: choose which export buttons (PDF, HTML, Word, Excel) are visible in the report toolbar
- PowerPoint slot reserved in Settings with "coming soon" label
- Disabling PDF grays out the PDF section order panel automatically
- Preferences synced cross-browser via ADO Extension Data
- 6 new Vitest tests (EV-01 to EV-06)

---

## v2.7.4 — 2026-05-15

**Report framing fields \+ performance improvements**

- New reorderable PDF sections: Objectives, Entry criteria, Exit criteria
- Incident and Request reference fields added next to Change number in the form
- Flaky test include/exclude toggle (mirrors sections model)
- Stale-while-revalidate cache with skeleton loader for test plan loading
- AbortController-based cancellation for report generation
- Fix: French → English localisation for PDF section order labels
- Migration: existing settings upgraded silently, no data loss

---

## v2.7.0 — 2026-04-20

**Test Configurations (F-15)**

- Configuration column in Results and History tables (conditional — shown only when \> 1 config detected)
- All 4 exports (Excel, PDF, HTML, Word) include the Configuration dimension
- TC × Configurations matrix in PDF for plans with 2–3 distinct configurations
- New KPI: "Covered configurations" (X / Y defined in the project)
- Pass rate weighting option in Settings: by configuration or by test case

---

## v2.6.0 — 2026-03-10

**Traceability v2 redesign (F-16)**

- Traceability tab fully redesigned: Epic → User Story → Test Case → Bug hierarchy
- Two view modes: 3-column Table and Cards — preference persisted per user
- Status aggregation per US and Epic: OK / NOK / PARTIAL / NOT EXECUTED
- Status filter on the traceability view
- Automatic bug deduplication (bug linked to both TC and US appears once)
- "Untraced suites" block: suites without linked US, collapsible, with 4 new KPIs
- Traceability PDF section redesigned to match the new hierarchy

---

## v2.5.5 — 2026-02-01

- Multi-US and multi-Epic traceability: a test case now shows all linked User Stories and Epics
- "Step marking" metric KPI with executive summary recommendation
- Test Configurations multiselect filter
- TC × Test Configurations heatmap tab

---

## v2.5.1 — 2026-01-15

- Tree-based suite selector with Expand all / Collapse all controls
- Screenshots always present in PDF exports after restoring from history

---

## v2.5.0 — 2026-01-01

- Auto-complete search on test plans (filter by name and iteration)
- Current test point status prioritised over run history for reliable pass rate

---

## v2.4.0 — 2025-11-15

- Excel (.xlsx) export with 4 sheets: KPIs, Results, Traceability, Bugs
- Configurable run limit (100 / 200 / 500 / 1000)
- Chart.js charts in PDF rendered locally (offline-compatible)

---

## v2.2.6 — 2025-10-01

- User preferences synced cross-browser via ADO Extension Data
- All settings (templates, fields, language, colour, thresholds) tied to your ADO account

---

## v2.2.0 — 2025-09-01

- Form templates: save, load and delete named configurations
- Drag-and-drop PDF section reordering
- Custom executive summary text override
- CSV history export
- Per-report notes (500 characters)
- Storage usage indicator

---

## v2.1.0 — 2025-08-01

- Regex validation per field (blocking or warning mode)
- PDF signature page with configurable boxes, pre-fill from form, sign line

---

## v2.0.0 — 2025-07-01

- Run attachments: PNG/JPEG embedded inline in PDF and HTML
- Field configuration: hide, rename, or add custom fields
- Configurable environment values
